HHG Phase 3a 一次性迁移工具

按顺序点 1 → 2 → 3。每步完成后看输出,有问题不要继续。

1检查环境配置

确认 Netlify 环境变量都设置好了。会显示当前的 LARK_BASE_TOKEN / LARK_CRM_TABLE_ID 等。
点击按钮开始

2给 CRM 表加「成交时间」字段

Phase 3a 客户成交时需要记录成交日期。这步会调用 crm_add_field action,在当前 LARK_CRM_TABLE_ID 指向的表上加一个名为「成交时间」的 datetime 字段。

⚠️ 如果字段已经存在会报错 FieldNameExist,这是正常的,跳到下一步即可。
点击按钮开始

3迁移 CRM 状态:发房源 → 已发房源等回复

Phase 3a 把客户状态从 5 个拆成 6 个(加了「待发房源」「已发房源等回复」)。
所有当前状态 = 发房源 的客户,会被改成 已发房源等回复。
Admin 之后可以手动把还没发的客户拖回到 待发房源。

⚠️ 这个操作不可逆,但只改状态字段,不删数据。建议先在 dev 环境跑(LARK_CRM_TABLE_ID 应该是 dev 表 tbl0NK9b4QRi7Plp)。

点击按钮开始

4创建「客户-房源推荐_测试」新表

在同一个 Base 下新建一张表(21 个字段)。完成后会返回新表的 table_id。

注意:建完表之后,你需要把 table_id 加到 Netlify dev 环境变量 LARK_REC_TABLE_ID,然后重新触发 dev 部署(Netlify → Deploys → Branch Deploy: dev → Retry with latest branch commit)。

⚠️ 重复点击会创建多张同名表。如果第一次失败请先去 Lark Bitable 里手动删除新建的表再重试。

点击按钮开始

5验证新表字段(配完环境变量后再执行)

完成 Step 4 + 配好 LARK_REC_TABLE_ID + 重新部署 dev 后,点这个验证。会列出新表的所有字段,核对字段名是否和后端代码一致。
点击按钮开始
